### useId Hook
- 컴포넌트 내부에서 자동으로 고유한 ID를 생성
- 렌더링마다 변하지 않는 DOM용 고유 ID를 만들기 위한 훅
```jsx
const id = useid();

<label htmlFor={id}>이메일</label>
<input id="email" />
```


### useEffect Hook
- 렌더링 이후에 실행해야 하는 side effect를 처리 하는 훅
- side effect : 화면 렌더링과 직접 관계 없는 작업 또는 외부와 상호작용하는 코드로 본래 목적 외에 발생하는 부수적인 작용
```jsx
useEffect(() => {
    실행 할 코드
}, [의존성]);
```
콜백 함수로 실행 할 코드를 정하고 의존성 배열로 언제 실행 할지 결정
- 빈 배열: 컴포넌트 마운트 시 1번만 실행
- 의존성 배열: 상태 변화에 따라 실행

### 컴포넌트 생명 주기
- Mount(생성) → Update(변경) → Unmount(제거)
```
컴포넌트 생성
  ↓
렌더링
  ↓
useEffect 실행 (Mount)
  ↓
state / props 변경
  ↓
렌더링
  ↓
useEffect 실행 (Update)
  ↓
컴포넌트 제거
  ↓
cleanup 실행 (Unmount)
```

### useEffect Hook 사례
 - API 호출하기 
 - 타이머 설정하기
 - 실시간 이벤트 처리하기 (이벤트 등록/해제)
 - 자동 저장 기능 구현하기 (localStorage)
 - 실시간 통신 기능 구현하기 (WebSocket)
